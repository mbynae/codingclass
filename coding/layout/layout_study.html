레이아웃 만들기


레이아웃을 만드는 방법에는 크게 3가지가 있습니다.

01. float 방식
02. flex 방식
03. grid 방식


각각의 방식에는 장단점이 있으며 상황에 따라 선택하여 사용하는게 좋습니다.

01. float 방식

float 방식은 기본적인 문서 배치의 흐름을 벗어나 페이지의 왼쪽이나 오른쪽으로 이동해 정렬하는 방식으로 clear을 통해 플로팅을 해제 하므로써 원하는 위치까지만 적용할 수 있습니다.
하지만 일부 익스플로어에서는 적용이 되지 않으며 복잡해질수록 clear을 적용하기가 힘들어지는 단점이 있습니다.
float 방식은 3가지로 나눠집니다.

02. flex 방식

flex 방식은 float 방식보다 간단하며 자주 쓰이는 방식입니다. flex 방식은 레이아웃을 만들 때 아우 유연하고 동적인 구조를 유지할 수 있으며 플렉스 박스의 속성을 잘 활용하면 미디어쿼리 없이도 다양한 디바이스에 대응하는 반응형 레이아웃을 만들 수 도 있습니다.
하지만 복잡한 구조에선 추가 작업이 필요하여 복잡해질 수 있는 단점도 존재합니다.

2-3. 심화 flex 방식

flex 방식은 한번에 하나의 블록만 옮길 수 있기 때문에 여러개의 블록이 있을 경우 정리하기 힘들 수 있다는 단점이 있습니다.
이 경우, 옮겨야 할 블록들을 따로 묶어서 적용해야 제대로 정리가 가능합니다.

03. grid 방식

grid 방식은 부모 요소가 자식 요소를 컨트롤하여 레이아웃을 만드는 방식입니다. 만들기 위한 각 부분을 테이블 처럼 나눠서 구역별로 생성합니다.
적어야 할 코드가 위의 두 방식보다 많지만 부모 요소만 수정하면 모든 자식 요소가 수정이 되므로 복잡한 레이아웃을 만들때 유용합니다.
